Telescope Commands

Activate Telescope
- Options are [yes,no,sim]
- [yes] : activate socket connection between telescope tracker and time-master
- [no] : run software without any telescope input
- [sim] : activate simulated telescope data, this does not open a socket. Any selection for "scan strategy" is ignored
Scan Strategy
- Options are [1D Raster, 2D Raster, Pointing Cross, BowTie]
- 1D produces a linear scan in chosen coordinate
- 2D produces a box scan made from stacked linear scans in the chosen coordinate
- Pointing Cross creates 2 linear scans in perpendicular coordinates. (example one constant RA scan, one constant DEC scan)
- BowTie is a linear scan at constant elevation (will probably be deprecated)
Constant Coordinate System
- Options are [RA,DEC,AZ,ALT]
- Coordinate option will create fixed position along which telescope moves. (example, constant RA will increment along DEC coordinates)
Delayed Start
- How many seconds the MCE and KMirror will be running until the telescope script is activated
Time to Traverse Scan Length
- How long to traverse one linear pass from both extremes of the map
Map Len
- Parameter only used with 2D maps
- Defines the height of the map, perpendicular to axes of motion
Map Size
- Defines the length of the map along the axes of motion. Units can be given in arsec, arcmin, and degrees.
Angle of Map Offset
- A setting of "0" will mean constant motion along the axis perpendicular to the coord chosen in "Constant Coordinate System"
- Value will rotate the field N degrees off of axis perpendicular to "Constant Coordinate System"
Size of 2D Vertical Step
- For a 2D map, this defines how far above or below the last constant coordinate the telescope will step to start the next linear pass
Source Coord 1
- Coord 1 can be [RA,AZ]
- This is the center of the map, or on a source
- If coord in RA , please provide in [h:m:s]
Source Coord 2
- Coord 2 can be [DEC,ALT]
- This is the center of the map, or on a source
- Coord should be given in [deg:arcmin:arcsec]
Epoch of Observation
- Options are [B1950.0, J2000, Apparent]
- Provides epoch for given source coordinates. Apparent should be used for any objects within the solar system or those with high apparent motions
Object Catalog Name
- Name can be anything, unless it is in the solar system. Then choose arbitrary name. (example: if the Moon, use notMoon)

MCE Commands

Observer
- Please put your initials (eg. VLB)
- This is put in the netcdf data files
Active MCE's
- Options are [MCE0,MCE1,BOTH,MCE Sim]
- MCE0 and MCE1 only activate one at a time
- BOTH activates MCE0 and MCE1 simultaneously
- Sim does not activate either, but only provides randomly generated data from a normal distribution
Datamode
- Options are [Error,SQ1 Feedback,Raw,Filtered SQ1 Feedback,Debugging,Mixed Mode (25:7),Mixed Mode (22:10),Mixed Mode (24:8),Mixed Mode (18:14)]
- The definitions for these MCE programmable modes can be found at the UBC website
- https://e-mode.phas.ubc.ca/mcewiki/index.php/Data_mode
Readout Card
- "All" will activate all detectors connected to both MCE0 and MCE1
- All other options will select only one readout card connected to only one MCE
Frame Number
- Default is set to large value that will keep MCEs running for duration of a given scan
- This should not be changed unless stricly necessary
Heatmap Alpha
- This adjusts the alpha value for the rolling average calculation in the heatmap graphs
- roll_avg[b][c] = ((1-alpha)*roll_avg[b][c]) + (alpha * d[b][c])
- [b][c] indexing corresponds to a given row/col of the detector map
Delete Old Columns
- When changing the raw data view to different row/col combinations , setting "Yes" deletes older row/col data on graph
Time Interval
- How many seconds of MCE data you want to view on a screen at a time
Show MCE Data
- This will turn off the raw mce data graph, leaving the telescope and heatmap graphs running
